* Slots de ciblage                                                        :SCOPE:
  /Section critique — détermine quels Frames sont visibles dans _SCOPE de cet agent./
  /Ces slots DOIVENT être posés sur les Frames AVANT que cet agent soit actif./

** Stratégie retenue
   Choisir UNE des deux stratégies (ou les combiner si justifié) :

   - *Stratégie A — slot discriminant (valeur)* :
     un slot commun à plusieurs Frames, filtré par valeur dans CHERCHER.
     ~CHERCHER.filtre: '$_->{phase} eq "fiscale"'~
     → fmatch ramène trop de Frames, le grep réduit ensuite.
     Efficace si les valeurs sont bien distribuées ET le volume de Frames global est petit.

   - *Stratégie B — slot de présence (existence)* :
     un slot propre à cet agent, posé uniquement sur les Frames qu'il doit traiter.
     ~CHERCHER.attribut: besoin_agent_xxx~ (sans filtre)
     → fmatch retourne directement et seulement les Frames concernés.
     Préférable dès que le volume global de Frames est grand (évite l'explosion combinatoire).

   Stratégie retenue : <A | B | A+B>
   Justification      : <pourquoi ce choix pour ce domaine>

** Tableau des slots de ciblage
   | Slot                  | Stratégie | Posé par                   | Valeur / condition      |
   |-----------------------+-----------+----------------------------+-------------------------|
   | <slot_presence_agent> | B         | <agent N-1 ou feed>        | existence seule suffit  |
   | <slot_discriminant>   | A         | <initialisation corpus>    | eq '<valeur>'           |

** Contrat de pré-population
   Avant que cet agent soit actif, les Frames suivants DOIVENT déjà porter leur slot de ciblage :
   - Frame <X> → slot ~<slot_presence_agent>~ — posé par : <qui ?>
   - Frame <Y> → slot ~<slot_discriminant>~ = ~'<valeur>'~ — posé par : <qui ?>

   ⚠ Si ce contrat n'est pas respecté, _SCOPE retourne une liste vide → les règles ne tirent jamais.
