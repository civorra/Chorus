* Cohérence du pipeline
  - Slot ciblage agent 1 (qualification)  : =besoin_qualification= — posé par feed initial
  - Slot ciblage agent 2 (ossature)       : =besoin_ossature= — posé par qualification R04
  - Slot ciblage agent 3 (thermique)      : =besoin_thermique= — posé par ossature R07
                                            ET par qualification R05 (isolants/membranes)
  - Slot ciblage agent 4 (securite-feu)   : =besoin_securite= — posé par thermique R04
  - Slot ciblage agent 5 (conformite)     : =besoin_conformite= — posé par securite-feu R05
  - Agent terminaison    : conformite pos 5 → règle R03 (Perl pur addrule) → solved()

  Note : les isolants et membranes (non structuraux) entrent dans le pipeline
  à l'agent thermique (pos 3) via besoin_thermique posé directement par qualification (R05).
  Ils reçoivent ossature_ok absent (pas de vérification ossature pour ces types).

* Paramètres moteur recommandés
  Namespace : COB
  _MAX_CYCLES (par agent) : N_frames × N_règles_agent × 10 (calibré dynamiquement dans run.pl)
  _MAX_ITER (Expert global) : N_frames × 25 × 10 (heuristique : 25 règles max pipeline)
  Pattern obligatoire (bug Chorus::Expert->new() ignore ses args) :
    my $xprt = Chorus::Expert->new();
    $xprt->{_MAX_ITER} = $max_iter;
