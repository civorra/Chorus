#+AGENT: securite-feu
#+PIPELINE_POS: 4
#+RULES_DIR: rules/securite-feu

* Domaine
  Vérification des exigences de sécurité incendie (§5) :
  performance REI minimale selon le type de bâtiment (§5.1),
  réaction au feu du parement intérieur (§5.2),
  épaisseur de plaque de plâtre requise (§5.3).
  S'applique à tous les éléments ayant passé le thermique.

* Slots de ciblage
  Stratégie B — slot de présence.
  Slot ciblé : =besoin_securite=
  Posé par : agent thermique (R04).

  | Slot           | Posé par  | Valeur attendue |
  |----------------+-----------+-----------------|
  | besoin_securite | thermique | 1               |

  Contrat pré-population supplémentaire (feed) :
    - =collectif=       : 0 ou 1 (bâtiment collectif)
    - =rei_atteint=     : 30, 45 ou 60 (REI atteint par la paroi)
    - =classe_reaction= : 'A1', 'A2', 'B', 'C', etc. (Euroclasses NF EN 13501-1)
    - =masse_surfacique_kgm2= : masse surfacique du parement kg/m²
    - =e_ba_mm=         : épaisseur totale de plaque de plâtre en mm

* Pipeline E/S
  Entrants : =besoin_securite=, =type_element=, =collectif=,
             =rei_atteint=, =classe_reaction=, =masse_surfacique_kgm2=,
             =e_ba_mm=
  Sortants : =feu_ok= ('OUI' ou 'NON'),
             =raison_feu_ko= (texte si NON),
             =besoin_conformite= (1)

* Ontologie
  - REI minimal requis (§5.1) :
      collectif = 0 (maison individuelle) : REI 30
      collectif = 1 (bâtiment collectif)  : REI 60
  - Réaction au feu parement (§5.2) : A1 ou A2-s1,d0 minimum → class rank ≤ A2
    (classes admissibles : A1, A2)
  - Masse surfacique minimale parement (§5.2) : 8 kg/m²
  - Épaisseur plaque de plâtre (§5.3) :
      REI 30 (non collectif) : e_ba ≥ 13 mm
      REI 60 (collectif)     : e_ba ≥ 26 mm

* Catalogue des Frames
  Tous les Frames du pipeline reçoivent la vérification feu.
  Les slots fire-safety (rei_atteint, classe_reaction, masse_surfacique_kgm2, e_ba_mm)
  sont fournis par le feed pour chaque élément.

* Dictionnaire des slots
  | Slot                  | Type   | Propriétaire  | Description                         |
  |-----------------------+--------+---------------+-------------------------------------|
  | besoin_securite       | int    | thermique     | Marqueur présence (=1)              |
  | collectif             | int    | feed          | 1 si bâtiment collectif             |
  | rei_atteint           | int    | feed          | REI effectif atteint (30/45/60)     |
  | classe_reaction       | string | feed          | Euroclass réaction au feu (A1, A2…) |
  | masse_surfacique_kgm2 | float  | feed          | Masse surfacique parement kg/m²     |
  | e_ba_mm               | float  | feed          | Épaisseur totale plaque de plâtre mm|
  | feu_ok                | string | securite-feu  | 'OUI' ou 'NON'                      |
  | raison_feu_ko         | string | securite-feu  | Texte raison KO incendie            |
  | besoin_conformite     | int    | securite-feu  | =1 pour agent conformite            |

* Catalogue des règles
  R01-rei              — vérifie rei_atteint ≥ rei_min(collectif) (§5.1)
  R02-reaction-feu     — vérifie classe_reaction est A1 ou A2 ET masse ≥ 8 kg/m² (§5.2)
  R03-epaisseur-ba     — vérifie e_ba_mm ≥ e_min(collectif) (§5.3)
  R04-pose-feu-ok      — pose feu_ok='OUI' si aucun KO détecté
  R05-pose-besoin-conformite — pose besoin_conformite=1 sur tous les éléments traités

* Helpers Perl
  ** rei_min($collectif) → int
     Source corpus : §5.1 — Performance REI minimale par type de bâtiment
     Retourne 30 si collectif=0, 60 si collectif=1.

  ** classe_reaction_ok($classe) → bool
     Source corpus : §5.2 — Classes de réaction au feu admissibles (Euroclasses)
     Retourne 1 si classe est 'A1' ou 'A2'.

  ** epaisseur_ba_min_mm($collectif) → int
     Source corpus : §5.3 — Épaisseur minimale plaque de plâtre par destination
     Retourne 13 si collectif=0, 26 si collectif=1.

* Contraintes & Pitfalls
  - R01/R02/R03 : EXCEPTION sur feu_ok (idempotence — ne pas appliquer si déjà posé)
  - R04 : poser feu_ok='OUI' seulement si feu_ok n'est pas encore défini
    (les règles R01–R03 posent feu_ok='NON' en cas de KO)
  - R05 : pose besoin_conformite=1 sur TOUS — cut() après pose
  - R01–R03 : retournent 0 si CONDITION non remplie (type spécifique si nécessaire)
  - Pitfall : une seule violation suffit à poser feu_ok='NON' — R04 ne doit
    se déclencher que si aucun slot raison_feu_ko n'est posé
