#+AGENT: conformite
#+PIPELINE_POS: 5
#+RULES_DIR: rules/conformite

* Domaine
  Synthèse de conformité globale (§6) : agrège tous les slots de validation
  posés par les agents précédents pour déclarer chaque élément CONFORME ou
  NON_CONFORME selon les conditions de §6.1.
  Pose le statut final et déclenche solved() quand tous les éléments
  ayant besoin_conformite défini ont leur statut posé.

* Slots de ciblage
  Stratégie B — slot de présence.
  Slot ciblé : =besoin_conformite=
  Posé par : agent securite-feu (R05).

  | Slot              | Posé par     | Valeur attendue |
  |-------------------+--------------+-----------------|
  | besoin_conformite | securite-feu | 1               |

* Pipeline E/S
  Entrants : =besoin_conformite=, =type_element=, =qualifie=,
             =ossature_ok=, =thermique_ok=, =feu_ok=, =collectif=
  Sortants : =statut_conformite= ('CONFORME' ou 'NON_CONFORME'),
             =raison_non_conformite= (texte explicatif si NON)
  Terminaison : =solved()= quand tous les Frames besoin_conformite
                ont leur statut_conformite posé

* Ontologie
  Conditions de conformité (§6.1) :
    Tout élément :
      + qualifie = 'OUI'
    Éléments structuraux (montants, lisses, entraits, chevrons) :
      + ossature_ok = 'OUI' ou 'NA' (entraits/chevrons exemptés ossature)
    Isolants et membranes :
      + thermique_ok = 'OUI'
    Si collectif = 1 :
      + feu_ok = 'OUI' (pour tous)
    Si collectif = 0 :
      feu_ok = 'OUI' requis aussi (REI 30 minimum — non collectif non exempté)

  Terminaison :
    Tous les Frames portant =besoin_conformite= ont =statut_conformite= défini
    → appeler $agent->solved() via règle Perl pure (addrule)

* Catalogue des Frames
  Tous les types du pipeline reçoivent le statut final (9 types).

* Dictionnaire des slots
  | Slot                  | Type   | Propriétaire | Description                         |
  |-----------------------+--------+--------------+-------------------------------------|
  | besoin_conformite     | int    | securite-feu | Marqueur présence (=1)              |
  | qualifie              | string | qualification| 'OUI' ou 'NON'                      |
  | ossature_ok           | string | ossature     | 'OUI', 'NON' ou 'NA'               |
  | thermique_ok          | string | thermique    | 'OUI', 'NON' ou 'NA'               |
  | feu_ok                | string | securite-feu | 'OUI' ou 'NON'                      |
  | collectif             | int    | feed         | 1 si bâtiment collectif             |
  | statut_conformite     | string | conformite   | 'CONFORME' ou 'NON_CONFORME'        |
  | raison_non_conformite | string | conformite   | Texte explicatif si NON_CONFORME    |

* Catalogue des règles
  R01-conforme     — pose statut='CONFORME' si toutes conditions §6.1 OK
  R02-non-conforme — pose statut='NON_CONFORME' + raison si au moins une KO
  R03-terminaison  — solved() quand tous les Frames ont leur statut (Perl pur addrule)

* Helpers Perl
  ** est_conforme($frame) → [bool, string]
     Source corpus : §6.1 — Conditions de conformité par type d'élément
     Vérifie qualifie, ossature_ok, thermique_ok, feu_ok selon le type.
     Retourne [1, ''] si CONFORME, [0, $raisons] sinon.

* Contraintes & Pitfalls
  - R01/R02 : EXCEPTION sur statut_conformite (idempotence absolue)
  - R01 et R02 mutuellement exclusifs (R01 si tout OK, R02 si au moins un KO)
  - R03 est une règle Perl pure (addrule) — jamais dans un YAML (risque boucle infinie)
    car elle fait un fmatch global sur besoin_conformite
  - Slot 'NA' (ossature) : ne bloque pas la conformité — est_conforme() gère ce cas :
      ossature_ok absent ou 'NA' → non bloquant pour isolants et membranes
  - _MAX_ITER global : calibré à N_frames × N_règles_total × marge (run.pl)
