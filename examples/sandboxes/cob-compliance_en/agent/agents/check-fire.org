* Domain

  Agent 4 of the TimberFrame pipeline.  Receives every ~TimberElement~ frame
  carrying the presence slot ~needs_fire = 1~, set by Agent 1 (R03) for all
  elements and optionally reinforced by Agents 2 and 3 guard rules.

  Fire safety requirements in this agent are sourced from three normative
  instruments:
  - *Building Regs Part B* / *BS EN 13501-2*: required fire resistance period
    (REI) by occupancy type (§5.1).
  - *BS EN 13501-1*: reaction-to-fire Euroclass and surface mass of internal
    lining (§5.2).
  - *Building Regs Part B §B3*: minimum plasterboard thickness to achieve the
    required REI (§5.3).

  The three fire-safety sub-checks are combined into a single rule (R01) to
  keep rule count minimal and reduce WM traffic.  A dedicated pass rule (R02)
  closes the phase and sets the downstream routing slot ~needs_compliance = 1~
  for Agent 5.

* Targeting Slots  :SCOPE:

** Chosen Strategy
   *Strategy B — presence slot.*
   The incoming presence slot is ~needs_fire = 1~, set by Agent 1 (R03) on
   ALL ~TimberElement~ frames regardless of element type.  Agents 2 and 3
   contain defensive guards that re-set ~needs_fire = 1~ if somehow absent.
   Strategy B is chosen for volume scalability.

** Targeting Slots Table
   | Slot             | Value    | Set by                    | Consumed by          |
   |------------------+----------+---------------------------+----------------------|
   | needs_fire       | 1        | Agent 1 (R03) / A2/A3 guard| This agent (R01,R02)|
   | fire_ok          | YES / NO | This agent                | Agent 5              |
   | needs_compliance | 1        | This agent (R02)          | Agent 5              |

** Pre-Population Contract
   ~chorus-feed~ MUST provide the following slots for all elements before
   engine activation:
   - ~collective~ — 0 (single dwelling) or 1 (multi-occupancy).  Mandatory.
   - ~fire_rei~ — measured or declared fire resistance period in minutes.
   - ~lining_reaction_class~ — Euroclass string: 'A1', 'A2', 'B', 'C', 'D',
     'E' or 'F'.
   - ~lining_surface_mass_kg~ — surface mass of lining in kg/m².
   - ~plasterboard_thickness_mm~ — nominal thickness of plasterboard in mm.

* Ontology

  Occupancy types and fire resistance requirements (§5.1 — Building Regs Part B
  / BS EN 13501-2):
  | collective value | Occupancy type           | REI minimum |
  |------------------+--------------------------+-------------|
  | 0                | Single occupancy dwelling| REI 30      |
  | 1                | Multi-occupancy building | REI 60      |

  Lining reaction-to-fire Euroclass ranking (§5.2 — BS EN 13501-1):
  : A1 (best) > A2 > B > C > D > E > F (worst)
  Minimum class required: A2 (i.e. A1 or A2 only are acceptable).

  Lining minimum surface mass (§5.2):  ≥ 8 kg/m²

  Plasterboard minimum thickness (§5.3 — Building Regs Part B §B3):
  | collective value | REI target | Minimum thickness |
  |------------------+------------+-------------------|
  | 0                | REI 30     | 12.5 mm           |
  | 1                | REI 60     | 25.0 mm (2 layers)|

  Euroclass ranking helper (internal):
  | Class | Rank |
  |-------+------|
  | A1    |    1 |
  | A2    |    2 |
  | B     |    3 |
  | C     |    4 |
  | D     |    5 |
  | E     |    6 |
  | F     |    7 |

* Rule Catalogue

** Rule: R01-check-fire-requirements

   *Intent*: Reject any element that fails at least one of the three fire-safety
   checks drawn from corpus §5.1, §5.2 and §5.3.  The three sub-checks are:
   (1) REI period meets occupancy-based minimum; (2) lining reaction class is
   A1 or A2 AND surface mass ≥ 8 kg/m²; (3) plasterboard thickness meets the
   minimum for the declared REI target.
   First-failure semantics: only the first failing sub-check is logged.

   #+BEGIN_SRC yaml
   RULE: R01-check-fire-requirements
   FIND:
     - FRAME: TimberElement
       WHERE:
         needs_fire: 1
   PREMISES:
     - |
       do {
           my $coll      = $frame->get('collective')               // 0;
           my $rei       = $frame->get('fire_rei')                 // 0;
           my $rei_min   = $coll ? 60 : 30;
           my $rc        = $frame->get('lining_reaction_class')    // 'F';
           my $mass      = $frame->get('lining_surface_mass_kg')   // 0;
           my $pb_thick  = $frame->get('plasterboard_thickness_mm')// 0;
           my $pb_min    = $coll ? 25 : 12.5;
           my $rc_rank   = _euroclass_rank($rc);
           ($rei < $rei_min)
               || ($rc_rank > 2 || $mass < 8)
               || ($pb_thick < $pb_min);
       }
   ACTION: |
     my $coll     = $frame->get('collective')                // 0;
     my $rei      = $frame->get('fire_rei')                  // 0;
     my $rei_min  = $coll ? 60 : 30;
     my $rc       = $frame->get('lining_reaction_class')     // 'F';
     my $mass     = $frame->get('lining_surface_mass_kg')    // 0;
     my $pb_thick = $frame->get('plasterboard_thickness_mm') // 0;
     my $pb_min   = $coll ? 25 : 12.5;
     my $rc_rank  = _euroclass_rank($rc);

     my $reason;
     if ($rei < $rei_min) {
         $reason = sprintf "REI %d < required REI %d (collective=%d) — §5.1",
             $rei, $rei_min, $coll;
     } elsif ($rc_rank > 2) {
         $reason = sprintf "lining class %s worse than A2 — §5.2", $rc;
     } elsif ($mass < 8) {
         $reason = sprintf "lining surface mass %.1f kg/m² < 8 kg/m² — §5.2", $mass;
     } else {
         $reason = sprintf "plasterboard %.1f mm < required %.1f mm (collective=%d) — §5.3",
             $pb_thick, $pb_min, $coll;
     }

     $frame->set('fire_ok', 'NO');
     $frame->delete('needs_fire');
     $self->log->warn(sprintf "R01: %s FIRE NON-COMPLIANT — %s", $frame->id, $reason);
   #+END_SRC

   *EXCEPTION*: If any of the input slots are absent, the PREMISE uses
   conservative defaults (rei=0, rc='F', mass=0, pb_thick=0) which will cause
   the rule to fire and reject the element.  Missing fire-safety data is treated
   as non-compliance.  Ensure all fire slots are provided by ~chorus-feed~.

** Rule: R02-fire-pass

   *Intent*: Approve every element still carrying ~needs_fire = 1~ after R01
   has run (i.e. all fire checks passed).  Sets ~fire_ok = 'YES'~ and
   ~needs_compliance = 1~ to route the element to Agent 5.

   #+BEGIN_SRC yaml
   RULE: R02-fire-pass
   FIND:
     - FRAME: TimberElement
       WHERE:
         needs_fire: 1
   ACTION: |
     $frame->set('fire_ok', 'YES');
     $frame->delete('needs_fire');
     $frame->set('needs_compliance', 1);
     $self->log->info(sprintf "R02: %s FIRE OK — routing to compliance check", $frame->id);
   #+END_SRC

   *EXCEPTION*: R02 MUST appear after R01 in the YAML rule file.  If R01 has
   cleared ~needs_fire~ on a rejected element, R02 will not fire for that frame.
   R02 also sets ~needs_compliance = 1~ for elements already rejected by R01
   (they do NOT reach R02), meaning ~needs_compliance~ is only present on
   fire-OK frames.  Agent 5 should handle missing ~needs_compliance~ on rejected
   elements gracefully (they are already terminated by previous agents).

* Perl Helpers

  None — no domain knowledge helpers for this agent.  The Euroclass rank
  mapping is implemented as an inline closure within the rule ACTION block.
  If the mapping needs to be shared with other agents in future, extract to
  ~TimberFrame::Domain::FireHelper~ and add to @EXPORT_OK.

  Reference: ~_euroclass_rank($class_str)~ returns 1 (A1) to 7 (F) or 99 for
  unknown classes (conservative — triggers rejection in R01 PREMISE check).

  Inline implementation used in R01:
  #+BEGIN_SRC perl
  my %_EC_RANK = (A1=>1, A2=>2, B=>3, C=>4, D=>5, E=>6, F=>7);
  sub _euroclass_rank {
      my ($c) = @_;
      return defined $c && exists $_EC_RANK{uc $c} ? $_EC_RANK{uc $c} : 99;
  }
  #+END_SRC

* Constraints & Pitfalls

  - *Rule ordering*: R01 must precede R02.  R02 is a catch-all; reversing order
    would approve all elements before R01 can reject them.

  - *Conservative defaults in R01*: Absent fire-safety slots default to the
    most non-compliant values (rei=0, class='F', mass=0, thickness=0).  This
    design ensures that data missing at ~chorus-feed~ level cannot silently
    produce a false pass.

  - *needs_compliance scope*: Only fire-OK elements receive ~needs_compliance=1~
    via R02.  Elements rejected in earlier agents (Agents 1–3) do not carry
    ~needs_compliance~ and are therefore never seen by Agent 5.  This is by
    design: only elements that have passed all phase-specific checks reach the
    global compliance aggregation.

  - *Plasterboard check scope*: §5.3 specifies plasterboard thickness as the
    means to achieve the required REI.  In this demo the plasterboard check is
    treated as an independent parallel condition alongside the REI declaration
    (R01 PREMISE checks both independently).  In a full implementation the REI
    check might subsume the plasterboard check — review with structural
    engineer before production use.

  - *Collective flag source*: The ~collective~ flag must be set once per
    project at the ~chorus-feed~ level and replicated to every frame.  It is not
    a per-element property in the physical sense, but is modelled as a frame
    slot here for simplicity.

  - *_MAX_CYCLES recommendation*: 100 elements × 6 slots × 5 agents × 10
    safety factor = *30 000*.  Set ~_MAX_CYCLES = 30_000~ in ~chorus-check~.
