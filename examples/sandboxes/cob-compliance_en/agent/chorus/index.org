* Overview

  This index covers the five-agent Chorus pipeline for the ~demo-cob-intro_en~
  sandbox.  The domain is timber-frame construction compliance (up to 2 storeys,
  wind zones 1–4, climate zones A/B/C).

  Corpus: ~corpus/001-timber-frame-intro-simul.txt~ (§1–§6).
  Sources: BS EN 338, BS EN 1995-1-1 (EC5), Building Regs Part L / Part B,
  BS EN 13501-1.

  All agents use *Strategy B* (presence slot targeting).  Recommended
  ~_MAX_CYCLES~: 30 000 (see each agent's Constraints section).

* Presence Slot Flow

  #+BEGIN_EXAMPLE
  chorus-feed
      │  sets needs_qualify=1 on all frames
      ▼
  Agent 1 — QualifyMaterial
      │  qualified=YES/NO
      │  sets needs_geometry=1  → load_bearing_stud, sole_plate, top_plate
      │  sets needs_thermal=1   → insulation, vapour_control
      │  sets needs_fire=1      → ALL elements
      ▼
  Agent 2 — CheckGeometry          Agent 3 — CheckThermal
      │  frame_ok=YES/NO               │  thermal_ok=YES/NO
      │  ensures needs_fire=1          │  ensures needs_fire=1
      ▼                                ▼
  Agent 4 — CheckFire  (receives ALL elements via needs_fire=1)
      │  fire_ok=YES/NO
      │  sets needs_compliance=1 on fire-OK elements only
      ▼
  Agent 5 — CheckCompliance
      │  compliance_status=COMPLIANT/NON_COMPLIANT
      │  clears needs_compliance
      ▼
  chorus-check addrule() termination
      solved() when no frame carries needs_compliance
  #+END_EXAMPLE

* Known Gaps & Future Work

  - *Non-load-bearing stud geometry*: Agent 1 does not route non-load-bearing
    studs to Agent 2 (~needs_geometry~ not set for this type).  If spacing or
    cross-section checks on non-load-bearing studs are required, update Agent 1
    R03 routing and Agent 2 R01/R02 PREMISES.

  - *Sole plate / top plate splice*: §3.4 is silent on splices in sole and top
    plates.  Agent 2 R03 only checks load-bearing studs.  Review corpus if scope
    expands.

  - *Rafter and joist geometry*: Rafters and joists receive ~qualified~ and
    ~fire_ok~ checks but no geometry check (no ~needs_geometry~ routing for
    these types).  A dedicated geometry agent or extension of Agent 2 would be
    required for span/depth ratio checks.

  - *Multi-failure reporting*: Current design uses first-failure semantics within
    each agent.  A ~failure_reasons~ accumulator slot could be added to collect
    all non-conformities per element before the compliance aggregation step.

  - *Contractor qualification (§1.2)*: The corpus §1.2 requirement (contractor
    must hold a recognised qualification) is not modelled as a YAML rule in this
    demo.  It could be implemented as a global pre-condition check in
    ~chorus-check~ before the engine is started, rather than as a per-element
    rule.

  - *_euroclass_rank extraction*: Currently implemented as an inline closure in
    Agent 4.  If reaction-to-fire class ranking is needed elsewhere, extract to
    ~TimberFrame::Domain::FireHelper~ and export via ~@EXPORT_OK~.
