#+TITLE: Agent CheckGeometry
#+AGENT: TimberFrame::Agent::CheckGeometry
#+PIPELINE_POS: 2
#+RULES_DIR: rules/check-geometry/

* Domain

  Agent 2 of the TimberFrame pipeline.  Receives ~TimberElement~ frames that
  have been routed from Agent 1 with the presence slot ~needs_geometry = 1~.
  The scope is limited to structural framing members: load-bearing studs, sole
  plates and top plates.

  The agent verifies three geometric criteria drawn from BS EN 1995-1-1 (EC5):
  stud centre-to-centre spacing (§3.1), minimum cross-section dimensions for
  the declared height/spacing combination (§3.2 for load-bearing studs, §3.3
  for non-load-bearing studs) and longitudinal continuity of load-bearing studs
  (§3.4).

  On full pass, the agent sets ~frame_ok = 'YES'~ and additionally ensures
  ~needs_fire = 1~ is present (in case Agent 1 did not set it, which should not
  occur but is guarded here for robustness).  On any failure it immediately sets
  ~frame_ok = 'NO'~ and clears ~needs_geometry~.

* Targeting Slots  :SCOPE:

** Chosen Strategy
   *Strategy B — presence slot.*
   The incoming presence slot is ~needs_geometry = 1~, set by Agent 1 (R03) on
   frames whose ~element_type~ is ~load_bearing_stud~, ~sole_plate~ or
   ~top_plate~.
   Strategy B avoids a full class scan at every inference cycle when frame
   volume is large.

** Targeting Slots Table
   | Slot           | Value    | Set by              | Consumed by          |
   |----------------+----------+---------------------+----------------------|
   | needs_geometry | 1        | Agent 1 (R03)       | This agent (R01–R04) |
   | frame_ok       | YES / NO | This agent (R04/R01–R03) | Agent 5         |
   | needs_fire     | 1        | Agent 1 (R03) / R04 | Agent 4             |

** Pre-Population Contract
   Agent 1 MUST have cleared ~needs_qualify~ and set ~needs_geometry = 1~ before
   Agent 2 activates.  The following slots must be pre-populated by
   ~chorus-feed~:
   - ~element_type~ — determines which section checks apply.
   - ~stud_spacing_mm~ — required for all stud-type elements.
   - ~section_b_mm~, ~section_h_mm~ — required for cross-section check.
   - ~clear_height_mm~ — required for load-bearing stud cross-section matrix.
   - ~has_longitudinal_splice~ — required for load-bearing studs (0 or 1).

* Ontology

  Element types handled by this agent (source: corpus §3):
  - ~load_bearing_stud~  — full geometry check (spacing + cross-section + splice)
  - ~sole_plate~         — spacing + cross-section check only (no splice rule)
  - ~top_plate~          — spacing + cross-section check only (no splice rule)
  - ~non_load_bearing_stud~ — minimum cross-section 38 × 63 mm (§3.3), spacing check

  Stud spacing limits (§3.1 — BS EN 1995-1-1 §5.1):
  | Constraint      | Value       |
  |-----------------+-------------|
  | Minimum spacing | 300 mm      |
  | Maximum spacing | 600 mm      |
  | Tolerance       | ± 5 mm      |

  Load-bearing stud cross-section matrix (§3.2 — BS EN 1995-1-1 §5.2):
  | Clear height ≤   | Spacing ≤   | b_min (mm) | h_min (mm) |
  |------------------+-------------+------------+------------|
  | 2700 mm          | 400 mm      | 38         | 89         |
  | 2700 mm          | 600 mm      | 38         | 140        |
  | 3000 mm          | 400 mm      | 38         | 140        |
  | 3000 mm          | 600 mm      | 44         | 140        |
  | > 3000 mm        | any         | 44         | 184        |

  Non-load-bearing stud minimum cross-section (§3.3):
  | b_min (mm) | h_min (mm) | Notes                 |
  |------------+------------+-----------------------|
  | 38         | 63         | Regardless of height  |

* Rule Catalogue

** Rule: R01-check-stud-spacing

   *Intent*: Reject any stud-type element whose centre-to-centre spacing falls
   outside the range [300 mm, 600 mm] (corpus §3.1 — BS EN 1995-1-1 §5.1).
   A tolerance of ± 5 mm is implicitly absorbed by the boundary conditions.

   #+BEGIN_SRC yaml
   RULE: R01-check-stud-spacing
   FIND:
     - FRAME: TimberElement
       WHERE:
         needs_geometry: 1
   PREMISES:
     - "defined $frame->get('stud_spacing_mm')"
     - "$frame->get('stud_spacing_mm') < 300 || $frame->get('stud_spacing_mm') > 600"
   ACTION: |
     $frame->set('frame_ok', 'NO');
     $frame->delete('needs_geometry');
     $self->log->warn(sprintf
         "R01: %s REJECTED — stud_spacing %.1f mm outside [300, 600] — §3.1",
         $frame->id, $frame->get('stud_spacing_mm'));
   #+END_SRC

   *EXCEPTION*: If ~stud_spacing_mm~ is undef the PREMISE short-circuits and
   the rule does not fire.  Frames where spacing is missing will proceed to R02
   and R03 unchecked — always provide this slot via ~chorus-feed~.

** Rule: R02-check-section

   *Intent*: Verify that the declared cross-section (b × h) satisfies the
   normative minima for the element type.  For load-bearing studs the applicable
   minimum is determined by the matrix in §3.2 (height × spacing), resolved via
   the helper ~_min_section_for_lb_stud~.  For non-load-bearing studs the fixed
   minimum is 38 × 63 mm (§3.3).  Sole plates and top plates are assimilated to
   the load-bearing stud matrix.

   #+BEGIN_SRC yaml
   RULE: R02-check-section
   FIND:
     - FRAME: TimberElement
       WHERE:
         needs_geometry: 1
   PREMISES:
     - "defined $frame->get('section_b_mm') && defined $frame->get('section_h_mm')"
     - |
       do {
           my $type = $frame->get('element_type') // '';
           my ($b_min, $h_min);
           if ($type eq 'non_load_bearing_stud') {
               ($b_min, $h_min) = (38, 63);
           } else {
               ($b_min, $h_min) = _min_section_for_lb_stud(
                   $frame->get('clear_height_mm') // 0,
                   $frame->get('stud_spacing_mm') // 0);
           }
           $frame->get('section_b_mm') < $b_min || $frame->get('section_h_mm') < $h_min;
       }
   ACTION: |
     my $type = $frame->get('element_type') // '';
     my ($b_min, $h_min);
     if ($type eq 'non_load_bearing_stud') {
         ($b_min, $h_min) = (38, 63);
     } else {
         ($b_min, $h_min) = _min_section_for_lb_stud(
             $frame->get('clear_height_mm') // 0,
             $frame->get('stud_spacing_mm') // 0);
     }
     $frame->set('frame_ok', 'NO');
     $frame->delete('needs_geometry');
     $self->log->warn(sprintf
         "R02: %s REJECTED — section %dx%d < required %dx%d mm — §3.2/§3.3",
         $frame->id,
         $frame->get('section_b_mm'), $frame->get('section_h_mm'),
         $b_min, $h_min);
   #+END_SRC

   *EXCEPTION*: If either dimension slot is absent the first PREMISE fails and
   the rule is skipped.  If ~clear_height_mm~ or ~stud_spacing_mm~ is missing
   for a load-bearing stud, ~_min_section_for_lb_stud~ applies the most
   conservative row (44 × 184 mm).

** Rule: R03-check-splice

   *Intent*: Reject any load-bearing stud carrying a longitudinal splice within
   the storey height (corpus §3.4 — BS EN 1995-1-1 §5.4).  Non-load-bearing
   studs, sole plates and top plates are excluded by the PREMISES type guard.

   #+BEGIN_SRC yaml
   RULE: R03-check-splice
   FIND:
     - FRAME: TimberElement
       WHERE:
         needs_geometry: 1
   PREMISES:
     - "$frame->get('element_type') eq 'load_bearing_stud'"
     - "($frame->get('has_longitudinal_splice') // 0) == 1"
   ACTION: |
     $frame->set('frame_ok', 'NO');
     $frame->delete('needs_geometry');
     $self->log->warn(sprintf
         "R03: %s REJECTED — longitudinal splice present on load-bearing stud — §3.4",
         $frame->id);
   #+END_SRC

   *EXCEPTION*: This rule only fires on ~load_bearing_stud~ elements.  If
   ~has_longitudinal_splice~ is absent it defaults to 0 (no splice assumed) and
   the rule does not fire.  Absent slot = no splice is a deliberate safe default.

** Rule: R04-geometry-pass

   *Intent*: Approve every element still carrying ~needs_geometry = 1~ after
   R01–R03 have run (i.e. no rejection has fired).  Sets ~frame_ok = 'YES'~
   and ensures ~needs_fire = 1~ is present for routing to Agent 4.

   #+BEGIN_SRC yaml
   RULE: R04-geometry-pass
   FIND:
     - FRAME: TimberElement
       WHERE:
         needs_geometry: 1
   ACTION: |
     $frame->set('frame_ok', 'YES');
     $frame->delete('needs_geometry');
     $frame->set('needs_fire', 1) unless defined $frame->get('needs_fire');
     $self->log->info(sprintf "R04: %s GEOMETRY OK — §3.1/§3.2/§3.3/§3.4", $frame->id);
   #+END_SRC

   *EXCEPTION*: R04 is the catch-all and MUST appear last in the YAML rule
   file.  If R01–R03 have cleared ~needs_geometry~ on a failed element, R04
   will not fire for that frame.

* Perl Helpers

** _min_section_for_lb_stud

   *Signature*: ~_min_section_for_lb_stud($clear_height_mm, $stud_spacing_mm) → ($b_min, $h_min)~

   *Description*: Returns the minimum required cross-section (b_min, h_min) in
   mm for a load-bearing stud (or sole/top plate) based on the storey clear
   height and stud spacing, according to the matrix in corpus §3.2 —
   BS EN 1995-1-1 §5.2.  Applies the most conservative row (44 × 184 mm) for
   any height exceeding 3 000 mm or for any unknown input.

   *Source*: corpus §3.2 — BS EN 1995-1-1 §5.2.

   *Called by*: R02-check-section.

   *@EXPORT_OK*: YES.

   #+BEGIN_SRC perl
   sub _min_section_for_lb_stud {
       my ($h, $sp) = @_;
       $h  //= 0;
       $sp //= 0;
       if ($h <= 2700 && $sp <= 400) { return (38,  89) }
       if ($h <= 2700 && $sp <= 600) { return (38, 140) }
       if ($h <= 3000 && $sp <= 400) { return (38, 140) }
       if ($h <= 3000 && $sp <= 600) { return (44, 140) }
       return (44, 184);  # height > 3000 mm — most conservative
   }
   #+END_SRC

* Constraints & Pitfalls

  - *Rule ordering*: R01, R02 and R03 must precede R04.  R04 is a catch-all;
    any frame still holding ~needs_geometry = 1~ at rule R04 is declared
    GEOMETRY OK.

  - *Sole plates and top plates*: The corpus only defines the cross-section
    matrix for studs (§3.2).  Sole plates and top plates are not explicitly
    listed in §3.2 but carry ~needs_geometry = 1~ from Agent 1.  They are
    assimilated to the load-bearing stud matrix as a conservative approach.
    If corpus §3 is expanded, add dedicated branches in R02 and a helper.

  - *Splice rule scope*: R03 only applies to ~load_bearing_stud~.  The corpus
    is silent on splices in sole and top plates; they are not checked here.

  - *Non-load-bearing stud routing*: Non-load-bearing studs are NOT routed with
    ~needs_geometry = 1~ by Agent 1 (R03 only sets the slot for
    load_bearing_stud, sole_plate, top_plate).  If the project scope expands to
    include non-load-bearing stud geometry, Agent 1 routing must be updated.

  - *_MAX_CYCLES recommendation*: 100 elements × 6 slots × 5 agents × 10
    safety factor = *30 000*.  Set ~_MAX_CYCLES = 30_000~ in ~chorus-check~.
